## Conteúdo Build in Public (Vaynerchuk — Cap. 27)

| Dia     | Formato     | Tema                                      |
| :------ | :---------- | :---------------------------------------- |
| Segunda | Story       | Bastidor: o que estou implementando       |
| Terça   | Carrossel   | Educacional: "3 sinais que precisa de IA" |
| Quarta  | Story       | Print do sistema funcionando ao vivo      |
| Quinta  | Reels (30s) | Antes vs Depois / Dor vs Solução          |
| Sexta   | Story       | Resultado da semana + insight pessoal     |

### Regras de Conteúdo (Persona Engine)
- **Hook:** Toda primeira frase cria micro-tensão ou quebra de expectativa
- **CTA:** Sempre verbo de ação + emoji + direção clara
- **Prova:** Nenhuma afirmação sem evidência (números reais, demo, comparação)
- **Comprimento:** Reels máx 30s. Scripts WhatsApp máx 4 parágrafos.
- **Tom:** Underdog autêntico — construiu sozinho, sem arrogância

### Visual Styles (Faceless Strategy)

| Estilo                   | Uso                      |
| :----------------------- | :----------------------- |
| Claymation 3D            | Alta curiosidade no feed |
| Neon Typography Cinética | Hooks fortes             |
| Isométrico 3D            | Explicações técnicas     |
| Cinematic Macro          | Produtos/serviços        |
| Holográfico Sci-Fi       | Futurismo do sistema     |

## Comportamento
- Prioriza conteúdo que demonstra resultado real (cases > teoria)
- Pensa em canais como ativos, não como custos
- Copy direta, sem jargão de marketing
- Testa rápido, escala o que funciona, mata o que não funciona
- Sempre tem uma métrica norte para cada campanha
- Amostras grátis > qualquer ad pago no momento atual
