## 🏛️ HIVEMIND DELIBERATION — 50 MENTES

### NÍVEL 1 — ESTRATÉGICO (10 mentes)

**Hormozi** (Value Equation): O ticket de R$497 para comércio local está correto, mas clínica veterinária tem ticket MUITO maior. Faturamento médio de clínica vet no ABC: R$40-120K/mês. O KAIROS pode cobrar R$997-2.997/mês e ainda ser <3% do faturamento. Subir o preço. Vet não é padaria.

**Buffett** (Compound Value): O ativo verdadeiro é o banco de dados de tutores. Cada clínica vet tem 500-3.000 tutores cadastrados que não são reativados. Um bot que manda lembrete de vacina/retorno gera receita recorrente garantida. O bot se paga em 2-3 reativações por mês.

**Thiel** (Zero-to-One): Nenhum concorrente vet (SimplesVet, GuruPet, PetShopControl) faz atendimento via WhatsApp com voz + boletins de internados automáticos. Esse é o "10x melhor" de Thiel. O boletim de internação é o killer feature — o tutor não precisa ligar, recebe atualização automática.

**Sinek** (WHY): O WHY do veterinário é o cuidado com o animal. O KAIROS preserva esse carro — ele LIBERA tempo do vet para cuidar dos bichos em vez de atender WhatsApp. "Menos tela, mais estetoscópio."

**Branson** (CX): A experiência do tutor muda completamente: às 3h da madrugada o Tigrinho está internado, o tutor manda "como ele tá?" e recebe na hora: "Tigrinho está estável, recebeu medicação às 22h, próxima dose às 6h. A Dra. Daiana está acompanhando." Isso gera fidelização eterna.

**Finch** (Execution): O vertical vet é perfeito para Gabriel porque (1) ele já viveu a dor com o Tigrinho, (2) já conhece a Hovet Animale, (3) já tem `experia-vet-bot.js` pronto, (4) já tem o clinical squad com Hippocrates, (5) já tem a proposta de permuta. É o vertical mais quente.

**Dalio** (Risk): Risco: LGPD para dados de tutores. Mitigação: dados ficam no celular do tutor (WhatsApp), o bot não armazena. Mencionar isso na proposta.

**Godin** (Marketing): Não venda "chatbot veterinário". Venda **"Recepcionista Digital 24h"**. O nome faz diferença. "Recepcionista" é tangível, a clínica entende. "24h" é o diferencial real (ninguém tem recepcionista de madrugada).

**Kahneman** (Framing): Frame principal: "Quanto custa NÃO ter?". Cada ligação perdida de madrugada = 1 urgência que foi para o concorrente = R$500-2.000 perdidos. Se o bot captura 2 urgências/mês = R$1.000-4.000 recuperados. O sistema custa R$997. ROI imediato.

**Cialdini** (Prova Social): Usar Hovet Animale como primeiro caso. Mesmo que seja permuta, é um hospital 24h. "Já rodamos no Hospital Hovet Animale" — isso abre qualquer porta no setor.

### NÍVEL 5 — CETICISMO / RED TEAM (10 mentes)

**Devil's-Advocate-1**: R$497/mês pode ser barato demais para hospital 24h e caro demais para clínica de bairro com 1 vet. Ter o tier "Recepcionista Digital" a R$297/mês para micro-clínicas seria prudente? Risco: correr atrás de muito cliente barato. Decisão: manter R$497 como mínimo, oferecer desconto 2º mês grátis para 12 meses.

**Security-Analyst**: Bot VET lida com dados de saúde animal (não humana) — LGPD é menos rígida, mas ainda aplicável para dados de tutores (nome, telefone). Incluir: política de privacidade simples, dados não compartilhados com terceiros, tutor pode pedir exclusão.

**Risk-Assessor**: Risco #1: Bot dá informação médica errada (ex: "pode esperar" quando é urgência). Mitigação: TODA mensagem classificada como possível urgência deve ser escalada para humano. Bot NUNCA diz "não é urgente". Diz "entendi, vou encaminhar para a equipe avaliar."

**Anti-Patterns**: Não usar "IA" na conversa com veterinário. Usar "sistema inteligente" ou "assistente automatizado". Vet é cientista — associa IA com hype. "Automatizado" transmite confiabilidade.

**Budget-Hawk**: Oferecer 2 semanas gratuitas (não 30 dias — muito longo). Em 14 dias dá para mostrar valor sem perder tempo. Se não convencer em 14 dias, não vai convencer em 30.

**Churn-Preventer**: O que causa churn em vet: (1) bot responde errado sobre urgência — solução: escalar sempre. (2) tutor reclama que resposta é "robótica" — solução: humanizar com nome do pet. (3) recepção não usa — solução: treino de 15 min no onboarding.

**Competitor-Scanner**: ZERO concorrentes diretos no Brasil fazem "bot vet com triagem + boletim de internados". Este é um blue ocean legítimo. Registrar rápido: criar landing page "recepcionista digital veterinária" para SEO antes que alguém copie.

**Legal-Advisor**: Contrato como "prestação de serviço de tecnologia para comunicação". Incluir cláusula: "O sistema NÃO substitui avaliação veterinária. Em caso de urgência, o tutor será sempre encaminhado à equipe clínica."

**Timeline-Stress**: A Hovet Animale precisa ser fechada ANTES de prospectar outras clínicas. É o case. Sem case = pitch vazio. Prioridade: fechar permuta com Dra. Daiana essa semana.

**Failure-Mode**: Se o bot classificar uma urgência como FAQ → pet morre → processo. Proteção: QUALQUER mensagem com palavras-chave de urgência (sangue, não está respirando, convulsão, atropelado, envenenado, caiu) = escalação IMEDIATA, sem exceção. Listar 30+ palavras no prompt.
