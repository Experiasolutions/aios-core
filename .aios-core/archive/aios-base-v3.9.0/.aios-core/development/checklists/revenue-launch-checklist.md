# Revenue Launch Checklist (Core)

> **Responsável:** @aios-master
> **Quando usar:** Antes de lançar qualquer operação de monetização em qualquer squad.
> **Princípio:** VII. Revenue-First (MUST)

## Revenue Gates
- [ ] **P&L projetado** — Margem ≥ threshold configurado
- [ ] **Juiz de Caixa** — `P(aceite) × margem × payback` calculado
- [ ] **Profit Firewall** — Brownout threshold definido e ativo
- [ ] **KPIs definidos** — T1C, Taxa Aceite, Margem, CAC, LTV mapeados

## Quality Gates
- [ ] **Critic-Gate** — Score ≥ 8 em rubricas aplicáveis
- [ ] **Arena** — Variantes testadas (se aplicável, mín. 3)
- [ ] **Kill rules** — Regras de kill definidas para ofertas/copy

## Security Gates
- [ ] **OPA Gates** — Policies verificadas
- [ ] **Privacy Budget** — Cota disponível para operação
- [ ] **C2PA** — Manifests gerados para criativos
- [ ] **Compliance** — Sem promessas proibidas

## Operational Gates
- [ ] **Tracking** — Métricas de rastreio configuradas
- [ ] **Runbook** — Plano de contingência pronto (brownout, kill, escalate)
- [ ] **Handoff** — Processo de handoff humano definido (se dor aguda)

## GO / NO-GO
- **Todos os gates ✅:** → **GO** 🚀
- **Qualquer gate ❌:** → **NO-GO** 🚫 (resolver primeiro)
